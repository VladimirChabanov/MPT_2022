# Практическое задание 1. Изучение базовых возможностей OC Linux

**Цель:** Освоить основные возможности консоли OC семейства Linux. Изучить базовый набор команд для работы с OC Ubuntu. Познакомится с фундаментальными концепциями используемыми в ОС семейства Linux.

## Задания к практической работе

### Установка

1. Скачайте и установите [VitrualBox](https://www.virtualbox.org/).  
   Для работы этой программы может потребоваться включить аппаратную виртуализацию в настройках BIOS вашего компьютера;
2. Создайте новую виртуальную машину и установите Ubuntu Server. Образ можно скачать на официальном [сайте](https://ubuntu.com/). В процессе установки можно оставить настройки по умолчанию. Инструкцию с комментариями по каждому шагу можно найти тут: https://losst.pro/ustanovka-ubuntu-server-20-04. На данный момент доступна версия 22.04 LTS, но в процессе установки нет серьёзных отличий от инструкции:
   - В процессе создания виртуальной машины установите галочку "Пропустить автоматическую установку", для того, чтобы иметь возможность выполнить конфигурирование сервера;
   - После шага 6, в указанной выше инструкции, пропущен выбор типа установки **Ubuntu Server** или **Ubuntu Server (minimized)**, второй вариант предусматривает использование в контейнерах и виртуальных машинах, не подразумевающих интерактивный вход и ручное управление. Это значит, что в такой системе отсутствуют утилиты ручного управления и если вы все-таки захотите администрировать ее руками, то вам придется развернуть ее до уровня Ubuntu Server. Поэтому здесь оставляем Ubuntu Server.
   - На шаге 7, настройки сетевого подключения менять не нужно.
   - На шаге 10, можно ничего не менять, а оставить Use an entire disk.
   - На шаге 11, придумайте уникальное имя пользователя (username) и установите простой пароль (например: 123).
3. После установки, перезагрузите сервер и залогиньтесь используя имя пользователя и пароль указанные в процессе установки.  
   Вся дальнейшая работа будет проходить в консольном режиме.
4. Чтобы получить справочную информацию по команде или программе можно воспользоваться командой `man`. Как правило вы будете получать ответ на английском языке. Чтобы прочитать ту же информацию на русском можно воспользоваться [сайтом](https://www.opennet.ru/man.shtml).

### Управление пользователями и правами

44. Ознакомьтесь с возможностями команды `sudo ` c помощью команды `man`;
45. Ознакомьтесь с возможностями команды `adduser ` c помощью команды `man`;
46. Создайте нового пользователя "user1" при помощи команды `adduser`. На все вопросы, кроме пароля можно отвечать нажатием Enter;
47. Ознакомьтесь с возможностями команды `passwd ` c помощью команды `man`;
48. Установите пользователю "user1" пароль "1234" с помощью команды `passwd`;
49. Аналогичным образом создайте пользователя "user2";
50. Ознакомьтесь с возможностями команды `su ` c помощью команды `man`;
51. Откройте 2-е дополнительные консоли ([Alt]+[F2] и [Atl]+[F3] соответственно). Залогиньтесь в обеих под своим основным пользователем.
52. Затем в одной из них перейдите под пользователя "user1", а во второй под пользователя "user2" при помощи команды `su`;
53. В первом терминале залогиньтесь под суперпользователем (root) при помощи `sudo su -`;
54. Из терминала "root" выполните команду `chmod 755 /home/user1 /home/user2`
55. Посмотрите права установленные для каталога `/home/user1/`
56. Из терминала "user2" попробуйте войти в каталог `/home/user1/`.
57. Ознакомьтесь с возможностями команды `chmod ` c помощью команды `man`;
58. Из терминала "user1" измените права доступа к каталогу `/home/user1/` на 700.
59. Посмотрите права установленные для каталога `/home/user1/`
60. Из терминала "user2" выйдите, а затем снова попробуйте войти в каталог `/home/user1/`.
61. Попробуйте перейти в `/home/user1/` из терминала с root;
62. Из терминала root создайте файл "file1.txt" с содержимым "made by root" в каталоге `/home/user1/`;
63. Из терминала "user1" попробуйте прочитать содержимое "file1.txt", а затем изменить его.
64. Ознакомьтесь с возможностями команды `chown ` c помощью команды `man`;
65. Из терминала root измените владельца файла "file1.txt" на "user1";
66. Из терминала "user1" попробуйте изменить "file1.txt", а затем удалите его.
67. Из терминала root создайте каталог `/var/share/`;
68. Установите ему права доступа 770;
69. Ознакомьтесь с возможностями команды `groupadd ` c помощью команды `man`;
70. Создайте группу "project" с помощью команды `man`;
71. Проверьте, что группа действительно создана при помощи просмотра файла `/etc/group`;
72. Ознакомьтесь с возможностями команды `usermod` c помощью команды `man`;
73. Добавьте пользователей "user1" и "user2" в группу "project" (как их дополнительную группу) при помощи команды `usermod` с ключами `-a` и `-G`. Ключ `-G`, заменяет доп. группы, в которых состоит, пользователь на указанные. Ключ `-a`, меняет поведение ключа `-G` на добавление групп вместо замены.
74. Ознакомьтесь с возможностями команды `chgrp` c помощью команды `man`;
75. Измените группу для каталога `/var/share/` на "project" при помощи команды `chgrp`;
76. Из терминала "user1" попробуйте перейти в каталог `/var/share/`. Сейчас это не должно получиться. Чтобы изменения вступили в силу, нужно перелогинится.
77. Используйте `exit` в терминалах "user1" и "user2", чтобы разлогиниться. Затем снова зайдите под этими пользователями.
78. Перейдите в каталог `/var/share/` в каждом терминале.
79. Из терминала "user1" создайте файл "file2.txt" с любым текстом.
80. Из терминала "user2" попробуйте изменить файл "file2.txt".
81. Проверьте права доступа к файлу. Как видно, несмотря на то, что каталог и оба пользователя состоят в группе "project" файлы по прежнему создаются с основной группой пользователя.
82. Изучите материал касающийся расширенных разрешений: https://habr.com/ru/post/469667/
83. В терминале "root" воспользуетесь командой `chmod g+s /var/share/`.
84. Из терминала "user1" создайте файл "file3.txt" с любым текстом и проверьте права доступа к файлу.
85. Ознакомьтесь с возможностями команды `gpasswd` c помощью команды `man`;
86. Удалите пользователей "user1" и "user2" из группы "project" при помощи команды `gpasswd` с ключом `-d`;
87. Ознакомьтесь с возможностями команды `groupdel` c помощью команды `man`;
88. Удалите группу "project" при помощи команды `groupdel`;
89. Ознакомьтесь с возможностями команды `deluser` c помощью команды `man`;
90. Удалите пользователей "user1" и "user2" с помощью команды `deluser`;
